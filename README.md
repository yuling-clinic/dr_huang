# 個人網站（小熊色）

小兒內分泌醫師的個人網站：門診時段、衛教文章、生活與旅遊紀錄。
奶油底色配深淺咖啡的日系雜貨風格，含視覺化後台與一鍵發布。

## 檔案結構

```
index.html         首頁（門診時段＋最新文章＋關於我）
articles.html      文章列表（可依分類篩選）
posts/*.html       每篇文章的獨立網頁 ← 發布時自動產生，Google 讀的是這些
admin.html         後台編輯（不會出現在導覽列，網址請自己記住）
css/style.css      全站樣式 ← 想改顏色改這裡最上方
js/render.js       共用邏輯
js/generator.js    靜態頁產生器（後台發布時用）
data/articles.json 所有網站內容
sitemap.xml        給 Google 的頁面清單（自動產生）
robots.txt         允許搜尋引擎收錄（自動產生）
images/            照片存放處
build.js           備用的本機建置腳本（平常用不到）
```

## 部署到 GitHub Pages

1. 在 GitHub 建立 repository，把全部檔案上傳（保持資料夾結構）。
2. Settings → Pages → Source 選 `main` 分支、根目錄 `/`，儲存。
3. 幾分鐘後即可用 `https://你的帳號.github.io/repo名稱/` 開啟。

網址可以之後再綁自己買的網域，隨時都能加。

## 🎨 想改顏色？

打開 `css/style.css`，**最上面**有一整區「配色設定區」，長這樣：

```css
:root{
  --cream:      #F3EADA;   /* 網頁底色：奶油 */
  --caramel:    #A97E54;   /* 焦糖：主要點綴色 */
  --coffee:     #6B4A2E;   /* 咖啡：標題文字 */
  ...
}
```

**只要改 `#` 後面那串色碼就好**，全站會一起變。前面的 `--名字` 和結尾的分號 `;` 不要動。

**查色號的工具**：
- Google 搜尋「color picker」→ 內建調色盤，最快
- coolors.co → 自動配一整組協調的顏色
- color.adobe.com → 上傳一張你喜歡的圖，自動抽出裡面的色碼

**小提醒**：改一個存一次、看一次，不要一次改十個。改之前先把原本那行複製一份放旁邊，改壞就貼回來。

## 後台編輯（admin.html）

### 首次設定（只做一次）

1. GitHub → Settings → Developer settings → Personal access tokens → **Fine-grained tokens**
2. 建立一個只能存取你網站 repo、權限為 **Contents: Read and write** 的 token
3. 打開 `admin.html`，填入帳號、repo、token，按「儲存連線設定」

Token 等於這個 repo 的鑰匙，只存在你自己的裝置，請勿分享或貼在公開場合。

### 日常流程

1. 開 `admin.html` → 按「從 GitHub 載入內容」
2. 編輯（見下方）
3. 按最下方「🚀 發布到網站」→ 約 1–2 分鐘後生效

### 寫文章

編輯器像 Word 一樣，選取文字後按工具列：

- **B / I** 粗體、斜體
- **大標 / 小標 / 段落 / 引言** 設定段落樣式
- **• 條列 / 1. 編號** 列點
- **📎 站內文章** → 連到自己的其他文章。先選取要變成連結的文字，按這顆，從清單挑一篇即可，完全不用管網址
- **🔗 外部連結** → 連到別的網站。選取文字後按這顆，貼上網址
- **🖼 插圖** → 選照片直接上傳（也可以直接貼上截圖）

### 系列文章

同一趟旅程想寫成一系列（例如多洛米蒂的行前準備 → 健行日記 → 素食紀錄）：

1. 每篇的「系列名稱」都填**一樣的名字**（例如「多洛米蒂健行」）
2. 「系列順序」填 1、2、3⋯⋯

這樣每篇文章底下就會自動出現整個系列的目錄，讀者可以一篇接一篇看下去，文章卡片上也會標示所屬系列。

## 注意事項

- 網站是公開的，**任何含病人資料的內容都不要放上去**（即使已去識別化）。
- 照片建議上傳前壓縮到寬度 1600px 內，載入才會快。
- GitHub Pages 免費方案：單一 repo 建議 1GB 內、每月流量約 100GB —— 對個人網站來說綽綽有餘。

## 🔍 讓 Google 搜尋得到你的文章

### 為什麼要特別處理？

如果網頁內容是靠 JavaScript 動態產生的，Google 收錄會慢又不穩。所以這個網站在你按「發布」時，會**自動為每篇文章產生一個獨立的 HTML 檔**（放在 `posts/` 資料夾），裡面的文字是寫死的——Google 一進來就直接讀得到，不需要執行任何程式。

同時也會自動產生：
- **sitemap.xml** — 告訴 Google 你有哪些頁面
- **robots.txt** — 允許搜尋引擎收錄（但擋掉後台）
- 每篇文章的 **title / description / og 標籤 / 結構化資料**（分享到 LINE、FB 時預覽卡片也會正常）

### 必做設定

在後台的「網站基本資訊」最上方，填入**網站網址**（例如 `https://yourname.github.io/my-website`）。這個欄位不填，發布會擋下來——因為 sitemap 和文章的正式網址都需要它。

### 提交給 Google（重要，只做一次）

網站部署好之後，Google 不一定知道你存在，要主動告訴它：

1. 到 **Google Search Console**（search.google.com/search-console）
2. 新增資源 → 選「網址前置字元」→ 貼上你的網站網址
3. 驗證擁有權（會給你一個 HTML 檔案下載，放進網站根目錄後上傳到 GitHub，再回去按驗證）
4. 驗證通過後，左邊選單「Sitemap」→ 提交 `sitemap.xml`

之後 Google 就會定期來抓你的新文章。Search Console 也能讓你看到有哪些人用什麼關鍵字找到你。

### 實際的期待值

- **搜尋你的名字** → 通常幾週內就找得到
- **搜尋一般衛教關鍵字**（如「兒童生長曲線」）→ 競爭很激烈，通常需要半年到一年的內容累積
- **長尾關鍵字**（如「多洛米蒂 不租車 交通」）→ 競爭小，反而可能更快有流量

身為醫師，你的衛教內容天然具備 Google 很重視的專業性與可信度（E-E-A-T）。網站上清楚呈現你的資歷與服務單位，對醫療類內容的排名有實質幫助——這部分已經寫進網站的結構化資料裡了。

## 📊 看網站流量

後台有「網站數據」區塊，支援兩個免費工具（都不使用 cookie、不追蹤個人身分，適合醫療類網站）：

### Cloudflare Web Analytics — 看有多少人瀏覽、哪幾篇最熱門

1. 到 dash.cloudflare.com 註冊免費帳號（**不需要**把網域轉過去）
2. 左側 Analytics & Logs → Web Analytics → Add a site → 輸入你的網站網址
3. 它給你的程式碼裡有一串 `"token": "xxxxx"`，把 xxxxx 貼進後台的欄位
4. 重新發布網站即生效

### Google Search Console — 看別人搜什麼關鍵字找到你

1. 到 search.google.com/search-console → 新增資源 → 網址前置字元
2. 驗證方式選「HTML 標記」，把它給的 content 值貼進後台欄位
3. 重新發布網站 → 回 Google 按「驗證」
4. 通過後，左側「Sitemap」提交 `sitemap.xml`

設定完成後，後台的數據區塊會出現直達儀表板的按鈕。

**沒填 token 就完全不會載入任何追蹤程式碼**——網站保持乾淨。
